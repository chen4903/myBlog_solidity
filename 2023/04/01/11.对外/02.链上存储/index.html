链上存储

简介

在solidity中，状态变量的存储布局是使用插槽slot的形式，一个slot大小32bytes，一共有2^256个slot。因为区块链是公开透明的，因此部署的智能合约的一切链上数据都是公开透明的，即使他被标注为private类型。

知道数据的实际存储位置，就可以找到实际的数据，在很多情形中都十分重要：

  • 查询任何链上数据
  • 假如有修改存储的函数，则需要找到目的存储空间的位置，这意味着合约可触及的范围内数据可改

解题示例

因为initial owner的密码存储到链上，因此我们可以查询到其密码。接下来让我们来找到它：

  1. initial owner存储于users[0]，即动态数组的第一个元素
  2. 动态数组位置的计算公式：slot A => length，slotV = keccak256(slotA) + index
  3. 那么users是位于slot 0，slot 0存储的就是动态数组的元素个数。实际元素的位置是keccak256(slot 0) + index。在本处，initial owner的存放的起始位置就是keccak256(0) + 0。
  4. 由于结构体里面有两个属性，属性是一个一个插槽依次排列的，因此initial owner的owner属性位于keccak256(0) + 0，password属性位于keccak256(0) + 1
  5. 接下来，我们需要用到ethersjs或者webjs进行查询，本处使用ethersjs

计算位置：将0作为参数输入，得到0x290decd9548b62a8d60345a988386fc84ba6bc95484008f6362f93160ef3e563，这是owner的位置，那么password就存放在下一个位置：0x290decd9548b62a8d60345a988386fc84ba6bc95484008f6362f93160ef3e564

变长数组

  • slot A => length：变量定义的插槽存放变长数组的元素个数
  • slotV = keccak256(slotA) + index：变长数组实际的元素存放的位置，index从0开始

以下是一个例子：我们声明的user位于slot 0，而user的第一个元素则需要计算：slotV = keccak256(slot0) + 0。（假如有第二个元素，则位于slotV = keccak256(slot0) + 1），经过计算得到数据实际存放位置：0x290decd9548b62a8d60345a988386fc84ba6bc95484008f6362f93160ef3e563

string&bytes

bytes 和 string 的编码是相同的。

  • 对于较短的值（短于32字节），数组元素与长度一起存储在同一个槽中。如果数据最多只有 31 字节长， 元素被存储在高阶字节中（左对齐），最低阶字节存储值 length * 2。

  • 对于存储数据长度为 32 或更多字节的字节数，主槽 p 存储 length * 2 + 1， 数据照常存储在 keccak256(p)。
