比特币交易

交易、消费、UTXO

  • 交易就是比特币持有者已授权把自己拥有的比特币转账给其他人

  • 花费是指签署一笔交易，转移一笔以前交易的比特币给以比特币地址所标示的新所有者

  • 未花费的交易输出（Unspent Transaction Output，UTXO），是还没有被下一个交易花费的交易输出
    • 不能分割，只能被整体消费
    • 单位是“聪” ， 1比特币= 1亿聪

交易输入输出

  • 交易就是比特币持有者已授权把自己拥有的比特币转账给其他人
    • 包含一个或多个输入
    • 包含一个或多个输出
    • 包含每一笔输入的所有权证明
  • 输出累加略少于输入，两者的差隐含为手续费

交易输入

  • 每个交易都有一个或多个输入，用来支付交易输出所需的价值

  • 交易输入通过引用将 UTXO标记为将被消费，并通过解锁脚本提供所有权证明

  • 交易输入包含三部分
    • 指向UTXO的指针，被记录在区块链中所在的交易的哈希值和序列号
    • 解锁脚本，用以满足设定在UTXO中的支出条件，数字签名和公钥
    • 序列号，相对时间锁定，在给定时间前该UXTO不能被花费

交易输出

  • 每个交易都会创造输出，生成一定数量的UTXO

  • 交易输出包含两部分

    • 转账额
    • 确定花费输出所需条件的加密难题
      • 锁定脚本/见证脚本/脚本公钥
      • locking script/witness script/scriptPubKey

交易的构成

  • Input：计划花费的UTXO的来源
    • txid : 引用的UTXO所在的那笔交易的ID
    • vout : 引用的UTXO所在交易的输出中的序号
    • scriptSig : 解锁脚本，包含付款人对本次交 易的签名()和付款人公钥()
  • Output：计划生成几个UTXO，分别给谁， 每个UTXO里面有多少BTC
    • value : 比特币数量
    • n : UTXO序号（从0开始）
    • scriptPubkey : 锁定脚本，包含命令和收款 人的公钥哈希（)

交易链

  1. Alice 从她朋友 Joe 那里用现金换了点比特币，那笔交易创建了一些被Alice 的密钥锁定的比特币
  2. 在她支付 Bob 咖啡店的新交易中使用了之前的交易作为输入，并以支付咖啡和找零作为新的输出
  3. 交易形成了一条链，最近交易的输入对应以前交易的输出
  4. Alice 的密钥提供了解锁之前交易输出的签名，因此向比特币网络证明她拥有这笔钱
  5. 她将买咖啡的这笔支付附加到 Bob 的地址上，实现“阻止”那笔输 出，明确指明要求是 Bob 签名才能消费这笔钱
