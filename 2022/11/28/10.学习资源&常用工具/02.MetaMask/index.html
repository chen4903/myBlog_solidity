MetaMask

简介

创建 MetaMask 是为了满足基于以太坊的安全和可用网站的需求。特别是，它处理帐户管理并将用户连接到区块链。ETH 是以太坊网络上的主要货币。但是，以太坊可以支持多种货币，而 ETH 不会有特权地位。

在以太坊上执行交易要么运行计算，要么存储数据。这会消耗网络 CPU 周期或存储空间。该费用由发起交易的账户支付。付款称为“gas”。Gas 目前使用 ETH 支付

购买、存储、发送和交换代币

MetaMask是一个浏览器插件和一个手机app，为您提供密钥库、安全登录、登录钱包和交易——所有的数字资产的一切

探索区块链应用

MetaMask 提供了连接到基于区块链的应用程序的最简单但最安全的方式。在新的去中心化网络上进行交互时，您始终处于安全环境中

文档：https://docs.metamask.io/guide/getting-started.html#basic-considerations

发送交易（工作原理）

交易是区块链上的一般行为。它们总是通过调用eth_sendTransaction方法在 MetaMask 中启动。它们可能涉及简单的以太币发送，可能会导致发送代币、创建新的智能合约或以多种方式更改区块链上的状态。它们总是由来自外部帐户的签名或简单的密钥对启动。

更加详细的例子

下面我们来对params的几个参数进行解读：

  • Nonce [ignored]
    • MetaMask忽略此参数。
    • 在以太坊中，每笔交易都有一个随机数。这样一来，每笔交易只能由区块链处理一次。此外，要成为有效交易，nonce 必须是0，或者之前编号的交易必须已经被处理（补充：如果交易一直失败，应该把nonce设置为最初那笔失败交易的nonce值，并且把gas值调高，再次发起交易）
    • 这意味着事务总是按给定帐户的顺序处理，因此增加随机数是一个非常敏感的问题，很容易搞砸，尤其是当用户使用同一个帐户与多个应用程序进行交互时，可能会跨越多个设备
    • 由于这些原因，MetaMask 目前不向应用程序开发人员提供任何方式来自定义其建议的交易的 nonce
  • Gas Price [optional]
    • 可选参数，最好用于私链。
    • 在以太坊中，每笔交易都指定了它所消耗的 gas 价格。为了最大化他们的利润，矿工将在创建下一个区块时首先选择具有更高gas价格的待处理交易。这意味着较高的 gas 价格通常会使您的交易处理得更快，但代价是更高的交易费用。请注意，这可能不适用于例如第 2 层网络，它可能具有恒定的 gas 价格或根本没有 gas 价格。
    • 在我们的默认网络上，MetaMask 允许用户在“慢”、“中”和“快”选项之间选择他们的 gas 价格
  • Gas Limit [optional]
    • 可选参数。对 Dapp 开发人员很少有用。
    • Gas limit 是一个高度可选的参数，我们会自动为它计算一个合理的价格。如果出于某种原因，您可能会知道您的智能合约会受益于自定义 gas 限制。
  • To [semi-optional]
    • 接收方地址，十六进制编码的以太坊地址。
    • 当没有to参数，但是又data参数的时候，就会发生合约创建
  • Value [optional]
    • 要发送的货币的十六进制编码值。在以太坊主网络上，这是以太 ，以wei计价，即1e-18以太币。
    • 请注意，以太坊中经常使用的这些数字的精度远高于JavaScript 数字，如果没有预料到，可能会导致不可预测的行为。因此，我们强烈推荐使用BN.js在操作用于区块链的值时。
  • Data [semi-optional]
    • 创建智能合约所必需的。
    • 该字段还用于指定合约方法及其参数
  • Chain ID [currently ignored]
    • 链 ID 当前由用户当前选择的网络在ethereum.networkVersion. 将来我们可能会允许一种方式同时连接到多个网络，此时此参数将变得很重要，因此养成现在包含的习惯可能会很有用。

API

MetaMask 将一个全局 API 注入到其用户访问的网站中window.ethereum。该 API 允许网站请求用户的以太坊账户，从用户连接的区块链读取数据，并建议用户签署消息和交易。提供者对象的存在表示以太坊用户

以太坊 JavaScript 提供程序 API 由EIP-1193指定

基本用法

对于任何重要的以太坊网络应用程序——也就是 dapp、web3 站点等——要工作，你必须：

  • 检测以太坊提供者 ( window.ethereum)
  • 检测用户连接到哪个以太坊网络
  • 获取用户的以太坊账户

提供者 API 是您创建功能齐全的 web3 应用程序所需的一切。

也就是说，许多开发人员使用便利库，例如ethers ，而不是直接使用提供程序。如果您需要比此 API 提供的更高级别的抽象，我们建议您使用便利库。

十六进制表示 十进制表示 网络

0x5 5 Goerli测试网络

其他的比如Ropsten\Rinkeby等，在今天（2022/10/23）已经不再用了，挂了

方法

ethereum.isConnected()

请注意，此方法与用户的帐户无关。

您可能经常会遇到“已连接”这个词，指的是 web3 站点是否可以访问用户的帐户。然而，在provider接口中，“已连接”和“已断开”是指provider是否可以向当前链发出 RPC 请求。

如果provider连接到当前链，则返回true，否则返回false

ethereum.request(args)

用于request通过 MetaMask 向以太坊提交 RPC 请求。它返回Promise解析为 RPC 方法调用结果的 a。

params和返回值将因 RPC 方法而异。在实践中，如果一个方法有 any params，它们几乎总是 type Array<any>。

如果请求因任何原因失败，Promise 将拒绝并，并返回Ethereum RPC Error

MetaMask 支持大多数标准化的以太坊 RPC 方法，此外还有一些其他钱包可能不支持的方法。有关详细信息，请参阅 MetaMask RPC API 文档。

连接

当 MetaMask的provider第一次能够向链提交 RPC 请求时，它会发出此事件。我们建议使用connect事件处理程序和ethereum.isConnected()方法来确定何时/是否连接了提供程序。

断开连接

如果 MetaMask 提供者无法向任何链提交 RPC 请求，它会发出此事件。通常，这只会由于网络连接问题或某些不可预见的错误而发生。

一旦disconnect发出，provider将不会接受任何新请求，直到重新建立与链的连接，这需要重新加载页面。您还可以使用该ethereum.isConnected()方法来确定提供程序是否已断开连接。

用户更改

eth_accounts每当RPC 方法的返回值发生更改时，MetaMask provider都会发出此事件。eth_accounts返回一个空数组或包含单个帐户地址的数组。返回的地址（如果有）是允许调用者访问的最近使用的帐户的地址。调用者由其 URL来源标识，这意味着具有相同来源的所有站点共享相同的权限。

accountsChanged每当用户公开的帐户地址更改时都会发出。

链切换

当当前连接的链发生变化时，MetaMask 提供程序会发出此事件。

所有的 RPC 请求都提交到当前连接的链上。因此，通过侦听此事件来跟踪当前链 ID 至关重要。

我们强烈建议在链更改时重新加载页面，除非您有充分的理由不这样做。

信息

MetaMask provider在收到一些应该通知消费者的消息时发出此事件。消息的种类由type字符串标识。

RPC 订阅更新是该message事件的常见用例。例如，如果您通过eth_subscribe来创建订阅，每个订阅更新都将被作为一个message事件（属于eth_subscription的一种类型）来发送

错误

MetaMask 提供程序抛出或返回的所有错误都遵循此接口：

您通常可以使用 errorcode属性来确定请求失败的原因。常见代码及其含义包括：

  • 4001：请求被用户拒绝
  • -32602：参数无效
  • -32603：内部错误

有关错误的完整列表，请参阅EIP-1193和EIP-1474.

这两个方法都是用于检查是否有权访问用户的账户。弃用原因：isEnabled和isApproved功能相同，除了isApproved是async。

我们推荐使用以下方法检查永固访问权限

  • 调用wallet_getPermissions这个RPC方法，检查eth_accounts访问权
  • 调用eth_accounts这个RPC方法和ethereum._metamask.isUnlocked()方法
    • 访问账户之前必须保证MetaMask是已经unlocked。如果通过eth_accounts返回的数组是空的，那么去查看以下MetaMask是不是被锁定了（调用isUnlocked()方法）

RPC API

MetaMask使用ethereum.request(args)方法来封装RPC API

该个API 基于所有以太坊客户端公开的接口，和一些方法（可能并不被其他钱包所支持）

以太坊 JSON-RPC 方法

对于 Ethereum JSON-RPC API，请参阅Ethereum wiki.

此 API 的重要方法包括：

  • eth_accounts
  • eth_call
  • eth_getBalance
  • eth_sendTransaction
  • eth_sign

权限方法

MetaMask 通过EIP-2255引入Web3 钱包权限. 在这个权限系统中，每个 RPC 方法要么是受限制的，要么是不受限制的。如果一个方法被限制，调用者必须有相应的权限才能调用它。同时，不受限制的方法没有相应的权限。其中一些仍然依赖于权限才能成功（例如，签名方法要求您拥有eth_accounts签名者帐户的权限），而另一些则需要用户确认（例如wallet_addEthereumChain）。

除了MetaMask Flask，唯一现有的权限是eth_accounts，它允许您访问用户的以太坊地址。将来会添加更多权限。

在底层，权限是简单的、与 JSON 兼容的对象，其中包含一些主要由 MetaMask 内部使用的字段。以下界面列出了消费者可能感兴趣的字段：

如果您有兴趣了解更多关于这种权限系统背后的理论，我们鼓励您查看EIP-2255.

eth_requestAccounts

此方法由EIP-1102指定. 它等效于已弃用的ethereum.enable()提供程序 API 方法。

它需要调用wallet_requestPermissions来获得eth_accounts许可。由于eth_accounts目前是唯一的权限，因此您现在只需要此方法。

此方法返回一个十六进制以太坊地址字符串的数组：string[]

发起请求的用户需要发送一个以太坊地址来识别。返回一个解析为单个以太坊地址字符串数组的 Promise。如果用户拒绝该请求，Promise 将拒绝并出现4001错误。该请求会导致出现 MetaMask 弹出窗口。你应该在用户发起请求的时候作出回应，例如按钮单击。在操作还没有完成之前，你应该总是拒绝再次发起请求

wallet_getPermissions

注意：这个RPC方法在移动端尚未可用

这个方法返回一个调用者权限的数组：Web3WalletPermission[]

获取调用者的当前权限。返回解析为对象数组的 Promise Web3WalletPermission。如果调用者没有权限，则数组将为空。

wallet_requestPermissions

注意：这个RPC方法在移动端尚未可用

这个方法返回一个调用者权限的数组：Web3WalletPermission[]

向用户请求给定的权限。返回一个 Promise，它解析为一个非空的Web3WalletPermission对象数组，对应于调用者的当前权限。如果用户拒绝该请求，Promise 将拒绝并出现4001错误。

该请求会导致出现 MetaMask 弹出窗口。您应该只请求权限以响应用户操作，例如按钮单击。

非权限方法

wallet_addEthereumChain

此方法由EIP-3085指定.

如果请求成功则返回null，否则返回错误

创建一个确认请求，将用户指定的链添加到 MetaMask。添加后，用户可以选择切换到该链。

与任何导致确认出现的方法一样，wallet_addEthereumChain 只能作为直接用户操作的结果调用，例如单击按钮。

MetaMask 严格验证该方法的参数，如果任何参数格式不正确，将拒绝请求。此外，MetaMask 会在以下情况下自动拒绝该请求：

  • 如果 RPC 端点不响应 RPC 调用。注意：调用是从后端发起的，而不是前端。如果您使用原始许可名单，它们将被阻止。
  • 如果RPC端点返回不同的链ID，这时eth_chainId将被调用
  • If the chain ID corresponds to any default MetaMask chains.

MetaMask 尚不支持使用没有 18 位小数的本地货币的链，但将来可能会支持。

wallet_switchEthereumChain

此方法由EIP-3326指定

如果请求成功则返回null，否则返回错误

如果错误码（error.code）是4902，说明请求的链没有被 MetaMask 添加，需要通过wallet_addEthereumChain 请求添加。

创建一个确认，要求用户切换到指定的链chainId。

与任何导致确认出现的方法一样，wallet_switchEthereumChain 只能作为直接用户操作的结果调用，例如单击按钮。

MetaMask 会在以下情况下自动拒绝请求：

  • 如果链 ID 格式错误
  • 如果指定链 ID 的链尚未添加到 MetaMask

wallet_registerOnboarding

作为 API 使用者，您不太可能必须自己调用此方法。有关更多信息，请参阅文档。

调用成功返回true，否则返回false

使用MetaMask将请求站点注册为创始人。将返回的Promise解析为true，如果有错误则拒绝。

此方法旨在在安装 MetaMask 之后，但在 MetaMask 载入完成之前调用。您可以使用此方法通知 MetaMask 您是那个被建议安装 MetaMask 的人。这允许 MetaMask 在用户引导完成后将用户重定向回您的站点。

除了直接调用此方法，您可以使用该@metamask/onboarding库

wallet_watchAsset

此方法由EIP-747指定

如果token被添加了就返回true，否则返回false

请求：用户在MetaMask追加的token。如果添加成功则返回true

大多数Ethereum钱包支持大多数的token，通常来自一个集中管理的token注册表。wallet_watchAsset使web3的开发者能够在运行的时候，询问他们的用户是否要追踪 他们的钱包。一旦成功被添加，token将很难与“那些通过被弃用方法添加的token”作区分，比如：一个中心化的注册表

移动端特有的RPC方法

wallet_scanQRCode

params:是可选的正则表达式

返回一个二维码对应的字符串

请求用户使用他们的设备摄像头扫描二维码。返回解析为字符串的 Promise，匹配以下任一：

  1. 正则表达式参数（如果提供）
