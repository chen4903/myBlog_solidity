07.合约初始化

发表于2022-12-24|更新于2022-12-24|03.solidity进阶

合约初始化

黄皮书定义

  1. 创建合约的交易被发送到以太坊网络。该交易包含输入变量，特别是：

    • msg.sender(s)：这是要创建新合约的直接合约地址或外部钱包的地址。

    • original transactor(o)：源头发起交易的钱包用户

    • 可用气体（g）：这是用户指定的，分配给合约创建的总气体。

    • Gas price (p)：这是单位 gas 的市场价格，它将被转换为Ether。

    • Endowment (v)：合约创建的时候转给合约的钱，单位Wei。默认值为零。

    • Initialization EVM code(i)：新合约中的constructor函数和初始化变量，采用字节码格式。

  2. 仅根据交易的输入数据，可以预先计算新合约的指定地址，create2操作码也可也做到。At this stage, the input state values are modified, but the new contract’s state is still empty.

  3. 然后，初始化代码在 EVM 中启动并创建一个实例合约。

  4. 在此过程中，状态变量发生变化，数据被存储，gas被消耗和扣除。

  5. 合约完成初始化后，它会将自己的代码和预先计算得到的地址相关联

  6. 最后，将剩余的 gas 、成功或失败的消息，异步返回给发送方s。

注意：在第 5 步之前，新合约地址不存在任何代码！

在黄皮书的脚注中：

During initialization code execution, EXTCODESIZE on the address should return zero, which is the length of the code of the account while CODESIZE should return the length of the initialization code (as defined in H.2

简而言之，如果您尝试在合约构建之前或期间检查智能合约的代码大小，您将得到一个空值。这是因为智能合约还没有制作出来，因此无法得知自己的代码量。
